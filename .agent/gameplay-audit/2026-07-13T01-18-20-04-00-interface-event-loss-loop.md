# Gameplay audit: interface event loss loop

**Timestamp:** `2026-07-13T01-18-20-04-00`

## Summary

Selection, field-change and action-request events are generated by interface domains but are not available through ordinary game snapshots. Multiple commands can accumulate records with identical frame/elapsed values, after which the next tick silently clears them.

## Plan ledger

**Goal:** preserve causal gameplay events independently from render cadence.

- [x] Identify interface event producers.
- [x] Confirm no event identity or command correlation.
- [x] Confirm command accumulation and tick deletion.
- [x] Confirm ordinary consumers cannot read the records.
- [ ] Add event journal and gameplay fixtures.

## Reachable sequence

```txt
command A emits event A at frame F
command B emits event B at frame F
no snapshot exposes A or B
next tick clears both
game state remains changed
event-driven consumer sees neither event
```

## Consequences

- Audio, VFX, telemetry or persistence added later can miss accepted actions.
- A consumer cannot distinguish duplicate events from separate commands.
- Replay cannot reconstruct event order from frame and elapsed alone.
- Event loss is silent and has no typed result.
