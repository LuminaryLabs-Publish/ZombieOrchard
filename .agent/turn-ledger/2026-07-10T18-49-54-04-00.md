## Findings

- All gameplay domains are created before the user starts a run.
- The page starts an uncancelled RAF loop immediately.
- One fixed `1/60` simulation step runs per RAF callback.
- Simulation speed therefore follows display refresh rate.
- All domains tick regardless of the active screen.
- Pause is only a screen transition and does not stop gameplay.
- Play and New Game do not create or reset session state.
- Outcome -> Title rebounds to Outcome because `ended` remains true.
- `GameHost.tick()` can inject extra ticks while the automatic loop is active.
- RAF, listener, renderer, domain, and host disposal are unowned.
- Existing deterministic scenario and Market transaction gaps remain downstream.
