## Findings

1. Orchard startup consumes global randomness while filling 26 apples.
2. Each apple requires a random tree, random id, two random offsets and a random rarity decision.
3. Every successful collection refills through the same global source.
4. Every night tick performs a global random spawn trial.
5. A successful spawn consumes additional random angle and id draws.
6. Apple activity therefore perturbs later pest outcomes.
7. Display cadence currently changes pest trial count and random cursor advancement.
8. No run seed, stream id, cursor, draw result, committed-tick receipt or replay journal exists.
9. Random ids prevent canonical replay comparison.
10. Snapshots expose outcomes but not random causality.
